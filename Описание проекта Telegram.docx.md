

**Описание проекта Telegram-бота «TouristNN»**

**1\. Цель проекта**  
Проект представляет собой интеллектуального Telegram-бота, который помогает пользователю составить **идеальный пешеходный маршрут** по туристическим местам Нижнего Новгорода. Бот анализирует интерес пользователя, узнаёт, сколько времени он готов посвятить прогулке, получает геопозицию — и на основе актуальных данных из 2GIS и возможностей искусственного интеллекта **создаёт персональный маршрут с описанием достопримечательностей и советами для путешественника**.

**2\. Основная идея**  
Во время анализа инструментов для разработки мы столкнулись с ключевой проблемой: **модели GPT, обученные на статичных данных, не могут гарантировать актуальность информации о реальных объектах** — особенно в контексте новых заведений, парков или событий. Поэтому наша цель заключалась в том, чтобы **объединить точность картографических сервисов с интеллектом нейросети**, способной превращать сухие данные в живой, понятный и увлекательный маршрут для пользователя.

**3\. Этапы реализации**

**1\. Проблема актуальности данных**  
GPT-модели часто создают вымышленные объекты или используют устаревшую информацию. Мы решили, что нужно использовать **источник с подтверждёнными и регулярно обновляемыми данными**.  
**2\. Поиск источника достоверных данных**  
Мы протестировали несколько картографических API:

* **Яндекс API** — содержит нужные данные, но требует платного тарифа.  
* **2GIS API** — предоставляет бесплатный доступ для учебных проектов и имеет актуальную, структурированную базу данных по туристическим объектам.

**Выбор был очевиден — 2GIS API.**  
**3\. Работа с данными**  
Изначально планировалось загрузить полную базу интересных объектов, но мы отказались от этого решения, потому что:

* данные быстро устаревают;  
* объём базы слишком велик;  
* снижается производительность и точность.

В итоге мы реализовали **динамический поиск** — бот в реальном времени обращается к API 2GIS, находит все релевантные места поблизости и использует их для построения маршрута.  
**4\. Расчёт времени и маршрута**  
Мы рассматривали возможность использовать API маршрутизации 2GIS для расчёта точных пешеходных путей, но этот подход оказался **избыточным для поставленной задачи**. Нам не требовалось строить детальные треки с координатами и поворотами — достаточно было **оценить время между точками**. Кроме того, API маршрутизации 2GIS возвращает большой объём технических данных (геометрии, сегменты пути, параметры навигации), что усложняло бы код и делало работу бота медленнее.

Поэтому было принято **оптимальное решение** — поручить расчёт времени и построение логики маршрута языковой модели **GigaChat**. Так мы сохранили простоту архитектуры, повысили скорость и обеспечили естественную подачу текста пользователю.  
**5\. Выбор языковой модели**  
Среди бесплатных GPT-аналогов был выбран **GigaChat от Сбера** — он:

* поддерживает русский язык на уровне нативного понимания,  
* предоставляет API-доступ для учебных проектов,  
* способен создавать логичные и эмоционально вовлекающие тексты маршрутов.

**6\. Интеграция компонентов**

* **Telegram Bot API** — взаимодействие с пользователем (ввод интереса, времени и локации).  
* **2GIS API** — поиск актуальных точек поблизости.  
* **GigaChat API** — генерация маршрута, описаний и советов.

**7\. Финальная логика работы**

1. Пользователь вводит интерес (например: *парки, театры, храмы*).  
2. Бот запрашивает длительность прогулки и геопозицию пользователя.  
3. Через 2GIS API бот находит все подходящие объекты рядом.  
4. Список найденных мест передаётся в GigaChat.  
5. GigaChat формирует структурированный текст маршрута, разделённый по шагам, и отправляет его пользователю.

**4\. Почему решение эффективно**

* **Актуальность:** 2GIS API гарантирует точные и обновляемые данные.  
* **Интеллект:** GigaChat создаёт понятные, “человеческие” описания маршрутов, делая прогулку интересной.  
* **Автоматизация:** пользователю нужно всего три действия — ввести интерес, время и отправить локацию.  
* **Надёжность:** минималистичный Python-код, система повторных запросов (retry), устойчивость к сетевым сбоям.  
* **Гибкость:** код легко масштабируется и адаптируется под другие города — сервис можно расширить не только на Нижний Новгород, но и на города области.

**Почему наш сервис лучший для пользователя**

* Он **соединяет интеллект нейросети и точность реальных карт**.  
* Пользователь получает **готовый, продуманный маршрут** без поиска вручную.  
* бот не требует регистрации, работает быстро и интуитивно понятен каждому.  
* **Универсальность:** сервис одинаково полезен как туристам, так и жителям города —всем, кто хочет открыть Нижний Новгород заново.  
* Каждый маршрут **уникален и персонализирован** под интерес пользователя.

**6\. Использованные технологии**

| Компонент | Назначение |
| ----- | ----- |
| **Python 3.13** | основной язык проекта |
| **Telebot (pyTelegramBotAPI)** | взаимодействие с Telegram |
| **2GIS API** | поиск и фильтрация актуальных объектов |
| **GigaChat API** | генерация маршрутов и описаний |
| **dotenv** | безопасное хранение токенов |
| **logging** | логирование и отладка |
| **requests** | выполнение HTTP-запросов |

**7\. Итог**  
**TouristNN** — это не просто бот, а пример того, как можно объединить **искусственный интеллект и реальные данные** для решения практической задачи. Он делает город ближе, прогулку — интереснее, а технологии — понятнее и доступнее. Проект демонстрирует, как современные API и нейросети могут работать вместе, создавая **реальную пользу для пользователя**.
